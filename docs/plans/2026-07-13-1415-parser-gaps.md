# Phase A — Parser Gap-Filling: Struct, Enum, Union, Typedef, Global Vars

**Date:** 2026-07-13
**Plan:** `docs/plans/2026-07-13-1415-parser-gaps.md`

## Scope
Fill remaining top-level declaration parser gaps: `struct`/`enum`/`union`
definitions, `typedef`, and proper global variable declarations. Add
corresponding codegen support so the emitted C output is valid.

## Files Touched

| File | Change |
|------|--------|
| `src/ast.h` | Add `NODE_UNION_DECL`, `NODE_ENUM_DECL`, `NODE_STRUCT_FIELD` |
| `src/parser.c` | Add `parse_struct_decl`, `parse_enum_decl`, `parse_union_decl`, `parse_typedef`; extend `parse_declaration`; replace stub var-decl path |
| `src/codegen.c` | Add `NODE_STRUCT_DECL`, `NODE_UNION_DECL`, `NODE_ENUM_DECL`, `NODE_TYPEDEF` handlers |
| `tests/parser/test_parser.c` | Add tests for struct/enum/union/typedef parsing |
| `tests/codegen/test_codegen.c` | Add tests for struct/enum/union/typedef emission |

## Architecture Impact
- `NODE_STRUCT_DECL` already exists but is dead code — now wired to parser/codegen
- Struct/union/enum bodies must be parsed inline within `parse_declaration` (before the
  consume-until-semicolon fallback), since these constructs embed `{` which the current
  stub would misinterpret
