# Phase A: Recursive-Descent Parser, AST, and C Code Generator

**2026-07-13 12:00**

## 1. Scope

Build the foundational compiler pipeline: lexer → parser → AST → C code generator. At the end of this phase, `c2 build file.c2 --emit-c` produces a valid `.c` file that compiles with `gcc -std=c23`.

No verification, no borrow checking, no drop injection, no synthesis.

## 2. Files touched

| File | Change |
|------|--------|
| `src/lexer.h` | **New** — Token type enum, Lexer struct, lexer API |
| `src/lexer.c` | **New** — Hand-written tokenizer for C23 + C² extensions |
| `src/parser.h` | **New** — Parser struct, parser API |
| `src/parser.c` | **New** — Recursive-descent parser building AST |
| `src/codegen.h` | **New** — Codegen API |
| `src/codegen.c` | **New** — AST-to-C pretty-printer |
| `src/main.c` | **Update** — Wire real pipeline, parse CLI flags, call lexer→parser→codegen |
| `tests/lexer/test_lexer.c` | **New** — Lexer unit tests |
| `tests/parser/test_parser.c` | **New** — Parser unit tests |
| `tests/codegen/test_codegen.c` | **New** — Codegen output comparison tests |

## 3. Architecture impact

- AST definitions in `ast.h` are used but unchanged.
- No new compiler passes (verifier, borrow checker, etc.) are introduced.
- `main.c` grows a proper pipeline dispatch; the stub messages are removed.
- The existing `error.h` error list is used for diagnostic collection.

## 4. Verification

1. `make build` compiles without errors or warnings.
2. Unit tests pass: `make test` runs all test binaries.
3. `c2 build examples/swap_bytes.c2 --emit-c` produces `_c2_out/swap_bytes.c`.
4. `gcc -std=c23 -Wall -Wextra _c2_out/swap_bytes.c -o /tmp/swap_bytes_test` compiles without errors.
5. The produced binary runs and produces correct output.
