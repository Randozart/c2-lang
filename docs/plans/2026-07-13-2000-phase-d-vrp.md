# Phase D: Value Range Propagation (VRP)

**DateTime:** 2026-07-13 2000
**Scope:** Implement a data-flow analysis pass that infers integer variable
ranges from assignments, loop boundaries, and branch conditions.

## Design

VRP walks the typed AST and populates `node->range` on variable references
with inferred `[lo, hi]` intervals. Ranges are computed by:

1. **Literal assignments**: `x = 42` → `x ∈ [42, 42]`
2. **For-loop headers**: `for (i = 0; i < N; i++)` → `i ∈ [0, N-1]` in body
3. **When-guard conditions**: `when x > 0` → `x ∈ [1, INT_MAX]` in guarded block

### Symbol table extension
Add a `ValueRange` field to `Symbol` to track the current inferred range for
each variable. The VRP pass updates this during its walk.

### Handling control flow
- For when guards with arrow: the then-branch gets the guarded range
- For for-loops: the body gets the loop-variable range
- Sequential code: ranges carry forward through assignments
- At join points (after if/when): range = union of predecessor ranges
  (simplified: just clear to unknown)
